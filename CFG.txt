Content: Custom Programming Language CFG & Sample Code Using the Language
Language name: p.0 (read as "P-zero") - Prototype Zero

A. CFG

(1) Nonterminals
		{ PROGRAM, 
		LINES, LINE, FULL_LINE, 
		DECL, PRINT, ASSIGN, 
		DECL_ITEMS, MORE_DECL_ITEMS, DECL_ITEM, 
		MORE_ASSIGN, 
		PRINT_PARTS, MORE_PRINT_PARTS, PRINT_PART, 
		EXPR, TERM, FACTOR }
(2) Terminals & Keywords
		Terminals: { "=", "+", "-", "*", "/", "(", ")", ",", ":", ">>>", "<<" }
		Keywords: { "int", "p" }  // Handled by lexer as separate token types:KW_INT,
		//  KW_PRINT
(3) Token classes:
		ID: letter (letter | digit | "_")*
		NUM: digit+
		STR: " (letter | digit | symbol | ESCAPE)* "  (where symbol = printable ASCII except ")
		ESCAPE: '\' ( 'n' | 't' | ' "' | '\' )
		NEWLINE: "\n"
		WHITESPACE: [ \t\r\f]+    // Ignored by lexer
		COMMENT: "//" ~NEWLINE*   // Ignored by lexer
(3) Start symbol: PROGRAM
(4) Production rules:

		// PROGRAM structure
		PROGRAM ->   ">>>"   LINES   "<<<"
		
		// STATEMENTS and line structure
		LINES ->   LINE   LINES   |   ε
		LINE ->   FULL_LINE   NEWLINE   |   NEWLINE   |   COMMENT   NEWLINE
		FULL_LINE ->   DECL   |   PRINT  | ASSIGN
		
		// Declaration rules
		DECL ->   KW_INT    DECL_ITEMS
		DECL_ITEMS ->   DECL_ITEM   MORE_DECL_ITEMS
		MORE_DECL_ITEMS ->   "," DECL_ITEM   MORE_DECL_ITEMS   |   ε
		DECL_ITEM ->   ID  |   ID "=" EXPR 
		
		// Assignment
		ASSIGN ->   ID "=" EXPR   MORE_ASSIGN
		MORE_ASSIGN -> ","   ASSIGN   |   ε
		
		// Print rules
		PRINT ->   KW_PRINT    ":"    PRINT_PARTS
		PRINT_PARTS ->   PRINT_PART MORE_PRINT_PARTS
		MORE_PRINT_PARTS ->   "," PRINT_PART MORE_PRINT_PARTS | ε
		
		// Print parts: a string literal or an expression
		PRINT_PART ->   STR | EXPR
		
		// Expression grammar (fully standard, left associative)
		EXPR ->   EXPR + TERM   | EXPR - TERM   | TERM
		TERM ->   TERM * FACTOR   |   TERM / FACTOR    | FACTOR
		FACTOR ->   NUM | ID | "(" EXPR ")"

	* Note: 
			// (for comments), \n, \t, \", & whitespaces will be handled by the lexer


B. Sample Code Using p.0:

>>>		// start

    int a, b, c											// declaration line must start with "int", no other "int"’s shall occur
    p: "Prototype 0 is cool!\n"							// prints a literal  string
    a = 1, b = 90, c = a + b							// assignment line is for assignments only
    int d = -3, e = 8									// implicit declaration with simultaneous assignment is allowed
    p: "sum of all = ", a + b + c + d + e, "a = ", a	// printing compound statements

>>>		// end
